## Key Decisions

### 1. ADR-0030: Reveal Codes is a co-equal editing surface
Reverses ADR-0006's read-only stance and ADR-0014's flow-layout choice. One model-coordinate selection shared by both panes; reverse-path from reveal edits to `InputEvent`s required; `FlowLayout` and chip views retired for the reveal pane; chapter-slicing re-homed to inline code editing. Orientation user-configurable (left/right/below), default right.

### 2. ADR-0031: Undo/redo carries the caret
Undo records `(changeSite, changeOffset)` in model coordinates alongside each mutation; restore uses document diffing to land the caret at the correct position after undo/redo. Prerequisite for the two-surface shared-undo contract in ADR-0030.

### 3. LT4 track closure
`setFigureCaption` is added as a first-class `InputEvent` (Option A, atomic reducer arm) consistent with the `insertBlock` precedent (ADR-0023); the AppKit rendering is an owned AppKit-only surface with the GUI smoke-check as the integration reality gate.
