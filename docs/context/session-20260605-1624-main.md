### Phase 1 — `app/` SwiftPM package + `DocumentBundle` + `DocumentModel`
New separate SwiftPM package at `app/` depending on `core/` via `.package(path: "../core")` (ADR-0011). Package identity of a path dependency is the directory name, so products are referenced as `.product(name: "UntitledCore", package: "core")`.

Targets: `UntitledShell` (library, pure Foundation + UntitledCore, file I/O only), `UntitledApp` (executable, `@main` SwiftUI/AppKit), `UntitledShellTests` (test suite). `mksample` helper target also present for generating test bundles.

`DocumentBundle` reads and writes a `.untitled` bundle (a directory holding `prose.txt` + `sidecar.json`, per ADR-0007), delegating format to core `serialize`/`parse`. `BundleError.missingProse` is the one unrecoverable rejection case.

`DocumentModel` (`@MainActor @Observable`) holds the live `Document` + `fileURL` + status string; `open()`/`save()`/`saveAs()` drive `NSOpenPanel`/`NSSavePanel`; `load(from:)` opens a known URL without a panel. `AppDelegate` sets activation policy `.regular` so a `swift run` executable presents a window rather than a dock-less background process.

4 real-path round-trip tests in `UntitledShellTests`: write→read equality, file creation, directory creation, missing-prose rejection pinned to `.missingProse`. All GREEN. Integration Reality satisfied: real temp-dir file I/O, no stub.

### Phase 2 — `displayProjection` in core + `Attribution` + `DocumentTextView` in shell
Core additions: `DisplayToken`/`DisplaySpan` (DisplayToken.swift) and `Document.displayProjection() -> [DisplayToken]` (Display.swift). Pure block-grained token stream covering `.paragraph`, `.sceneBreak`, `.setPieceLine`, and `.chapterStart`, with chapter-cut splicing including mid-paragraph offset splits that preserve italic across the cut boundary. 11 behavioral tests in DisplayTests.swift; core suite now 59 GREEN, 0 warnings.

Shell additions: `Attribution` (Attribution.swift, AppKit-only) maps `[DisplayToken]` → `NSAttributedString` — the closed typographic vocabulary: body text with paragraph indent, verse/epigraph centered + italic, scene-break `* * *` ornament, chapter headings, small-caps via OpenType feature. `DocumentTextView` (DocumentTextView.swift) is an `NSViewRepresentable` hosting a read-only TextKit 2 `NSTextView` (created with `usingTextLayoutManager: true`, per ADR-0003) inside an `NSScrollView`, wired into `ContentView` which calls `model.document.displayProjection()` and passes the result through `Attribution`.

### Bug diagnosed and fixed: TextKit 2 blank-page rendering
The `NSTextView` rendered a blank page despite the document loading 5 blocks. Two root causes, both fixed:

1. The text view was created at zero frame — its text container had zero width and laid out nothing. Fixed by setting `frame`, `minSize`, `maxSize`, and `textContainer.containerSize` at make time (sized to the scroll view's content area) with `widthTracksTextView = true`.
2. Setting content via `textContentStorage.attributedString = string` does not drive the layout manager to render. Fixed by editing the backing `textStorage` inside `contentStorage.performEditingTransaction { ... }`. This is the correct TextKit 2 content-setting path; the technique is documented in a comment in DocumentTextView.swift.

Diagnosed by temporarily logging string length and view/container sizes to stderr plus an env-var auto-open launch hook; both debug affordances removed after the fix.

The user visually confirmed the full document renders correctly: indented paragraphs, italic word, centered `* * *` ornament, centered italic verse line, bold centered chapter heading.

## Key Decisions

### 1. ADR-0011 — editor shell is a separate SwiftPM package
A new `app/Package.swift` declares the app and shell targets, depending on `core/` via a local path reference. This isolates AppKit/SwiftUI from `UntitledCore` mechanically (separate build graph, separate package boundary), upholds the dependency-direction rule (ADR-0002, rule 8), and avoids polluting the core package with platform conditions.

### 2. ADR-0012 — `displayProjection` emits pure `DisplayToken`s in core; attribution in the shell
Option A chosen: `UntitledCore` emits `[DisplayToken]` (plain values, no AppKit), and the shell converts to `NSAttributedString` in a dedicated `Attribution` module. Consistent with `revealProjection`/`RevealToken` (ADR-0004, ADR-0006); keeps `displayProjection` headlessly testable.
