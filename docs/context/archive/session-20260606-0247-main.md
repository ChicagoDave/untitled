## Key Decisions

### 1. blockQuote added to closed PresentationOverride vocabulary (ADR-0009 amendment)
The existing two overrides (`alignment` and `smallCaps`) were not expressive enough for the most common prose structural block (an indented quotation). `blockQuote` was added as a scoped, closed addition — not an open door. It carries a symmetric margin inset with leading alignment and renders in `Attribution.swift`. The ADR-0009 guard ("justify to reveal") was applied: the block type has an unambiguous, distinct visual presentation that justifies its existence in the vocabulary.

### 2. Wire-token codec promoted to public enum API (rule 8b)
The `PresentationOverride.token` / `init?(token:)` codec was previously buried in `Storage.swift` as private helpers. Because both the sidecar and the new template front-matter parser must decode the same token strings, promoting the codec to the enum's public surface is the co-located wire-type sharing rule applied directly: a single type definition that both decoders import, preventing drift.

### 3. insertBlock as a new pure reducer op (ADR-0023, Option A)
A new `InputEvent` case keeps the insertion atomic, headlessly testable, and consistent with the existing `applyInput` pure-function pattern. App-layer sequencing of existing ops (Option B) was rejected because it would have introduced ordering dependencies and could leave the document in a partial state if interrupted.

### 4. Palette built-ins scoped to Scene Break only
`EditorLayout` marks set-pieces (verse, epigraph, letter) as non-editable rendered regions. Inserting one via the palette would produce a block the writer could not immediately type into, violating the "editable block" promise that makes the palette useful. The scope decision defers set-piece palette entries until `EditorLayout` can mark palette-inserted set-pieces as editable.
