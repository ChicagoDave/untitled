### Two editor bugs found and fixed during live verification
1. **Popovers anchored at screen bottom** — `NSTextView.firstRect(forCharacterRange:)` returns `.zero` for a caret (empty selection range) under TextKit 2. Fixed by computing the caret rect from `textLayoutManager.enumerateTextSegments(in:type:options:)` + `textContainerOrigin`. Also fixed popover sizing: `NSHostingController.sizeThatFits` on a not-yet-displayed controller collapses the list; replaced with `sizingOptions = .preferredContentSize`.
2. **Typing below the last block was a dead zone** — `EditorLayout.modelPosition` clamped a caret past the last editable segment to `nil`; `insertText` received no position and dropped the keystroke. Fixed by clamping to the last editable segment in `modelPosition`, and `insertText` now starts a fresh paragraph when the caret lands below a non-empty last block.

## Key Decisions

### 1. Pivot: snippets via @-completion; bible via side panel (ADR-0021)
The original plan placed both snippet completion and bible peek under `@`, with a transient overlay for the peek. Mid-session the user redirected to a cleaner separation: `@` inserts reusable text snippets (datelines, epigraphs); the bible is a persistent, searchable side panel the writer consults voluntarily. This eliminates the peek overlay, flick-to-last, and `SceneReferences` annotation entirely, reducing surface area while delivering the same reference-lookup value.

### 2. BibleIndex and SnippetIndex in GalleyShell, not GalleyCore (ADR-0020)
Both indexes require file I/O (reading package subdirectories). Placing them in `GalleyCore` would introduce `FileManager` into the pure domain core, violating ADR-0002/ADR-0008. `GalleyShell` is the correct owner; it already holds `WorkspaceDocument` and `DocumentBundle`.

### 3. caret-rect computation under TextKit 2
`NSTextView.firstRect(forCharacterRange:)` is unreliable for a zero-length (caret) range under TextKit 2 — it returns `.zero`. The correct approach is to walk `NSTextLayoutManager.enumerateTextSegments` for the caret range, take the first segment's frame, and add `textContainerOrigin`. This pattern should be applied to any future popover anchoring in the editor.
