Fixed by maintaining one persistent `NSTextView` whose bound buffer is swapped in `updateNSView` (preserving first-responder) plus an explicit `makeFirstResponder` call on launch.

## Key Decisions

### 1. ADR-0015 — Unsaved buffer and last-buffer policy
Blank buffers created by Cmd-N have no `fileURL` and live in memory only. Close prompts only if `hasContent == true`. An empty blank is discarded silently. Closing the last buffer replaces it with a fresh blank (never quits). Rationale: matches standard macOS text editor behavior; avoids accidental quit from Cmd-W.

### 2. ADR-0016 — Workspace store in GalleyShell (fix (a) of the plan-review contradiction)
`WorkspaceModel` and `WorkspaceDocument` live in `app/Sources/GalleyShell/` (pure Foundation + GalleyCore, no AppKit). Only the panel-runner (`FilePanels`) and SwiftUI command/menu glue live in the `Galley` executable. Enforces ADR-0011's testability boundary.

### 3. One persistent NSTextView, not .id(currentIndex) rebuild
Swapping `DocumentTextView`'s bound `WorkspaceDocument` in `updateNSView` rather than rebuilding the `NSViewRepresentable` on each switch preserves first-responder and avoids the NSTextView teardown regression.
