### Design rethink (co-designed conversationally)

The palette showed only "Scene Break" in a new/blank buffer because `TemplateIndex` loaded only the per-project `templates/` folder â€” a project-scoped directory that does not exist until the project is saved. The design conversation surfaced three structural corrections:

1. **Templates are the writer's cross-project toolkit**, not this-novel's data. Unlike bible entries and snippets (which are story-specific), block templates (Epigraph, Dateline, Block Quote) belong in every project including new ones. This motivated a layered library with a built-in layer that is always present in-process.

2. **Dedication reclassified** from a styled block template to a section insert (same mechanism as Prologue/Epilogue). A dedication is a roled `ChapterCut` + fresh prose â€” not an italic single block. Its vertical centering on its own page is a typesetter concern, not an authoring concern.

3. **Prologue/Epilogue/Dedication/Chapter are section inserts, not block types.** They use a closed `SectionRole` enum on `ChapterCut` (ADR-0026, deferred to LT2), not bare title strings, so the typesetter can drive page layout from intent rather than parsing free-form text.

4. **Authoring/typesetting boundary established** (ADR-0024): Galley captures intent (roles, asset refs, prose content + structure); it never paginates, vertically centers, or renders final image placement. That belongs to a separate typesetting process consuming the `.galley` format. The format is the contract.

5. **Images parked** as a future closed-vocabulary `figure`/`image` block + `images/` package dir + placeholder in the editor; typesetter places/sizes. Not in this track.

### ADR-0024 â€” Authoring and typesetting are separate

Written to `docs/adrs/0024-authoring-and-typesetting-are-separate.md`. Records: (1) Galley is an authoring tool, not a typesetting tool; (2) the `.galley` format is the contract between authoring and typesetting; (3) output layout (vertical centering, new-page breaks, image placement) is deferred to a typesetting process; (4) the form of that process (in-app / CLI / separate tool) is left open. Consequence for `SectionRole`: roles are intent the typesetter consumes; Galley does not act on them at display time.

## Key Decisions

### 1. Templates are cross-project; bible/snippets remain project-scoped
The palette was broken for new buffers because templates were modeled as project data. The design rethink established that block templates are the writer's reusable toolkit (like a word processor's paragraph styles), while bible entries and snippets are story-specific. This distinction drives the layered library design and amends ADR-0022 without touching ADR-0020.

### 2. Built-in templates as in-code values, not bundled resource files
Shipping templates as `[BlockTemplate]` constants in `BuiltInTemplates.swift` (in `GalleyShell`) means no disk I/O, no parse errors, always present even before a project exists. The built-in layer uses the same `BlockTemplate` type as user/story templates â€” no parallel type.

### 3. Dedication = section insert, not a template
Reclassifying Dedication out of the template layer prevents the wrong mental model (a styled block) from being baked in. The correct model (a roled `ChapterCut` + fresh prose) defers output layout to the typesetter. This is a consequence of ADR-0024.

### 4. Merge lives in TemplateIndex, not in the palette
`BlockPalette.items(templates:)` already accepted a `TemplateIndex` and built its item list from it. Placing the three-layer merge inside `TemplateIndex.merged(...)` keeps the palette decoupled from how many layers exist and where they come from. The palette needed no changes.
