## Key Decisions

### 1. Phase 6 reverted — wrong UX model
The `ChapterCut.opener` decoration model coupled structure to chapter boundaries and surfaced it through a non-keyboard flow. It was inconsistent with the project's emerging keyboard-first principle and introduced a class of non-editable rendered structure. Superseded by the block palette track.

### 2. Keyboard-first is now a recorded project principle
All structure creation must be reachable by keyboard. Cmd-; is the palette chord (deliberate: not `>`, `#`, or `/`, which occur in prose). Recorded in Claude memory (`keyboard-first-writing-ux`); referenced in the new plan track.

### 3. User templates = closed vocabulary only (ADR-0009 preserved)
Templates may specify presentation only from the existing `PresentationOverride` enum (and a scoped `blockQuote` amendment if deliberated in BP1). No open style system. This decision keeps ADR-0009 load-bearing and prevents a future session from treating the template system as a styling escape hatch.
